无人机精准降落技术方案

发表于2025-04-10|更新于2025-04-10|技术博客
|总字数:6.5k|阅读时长:22分钟|浏览量:

精准降落简述

启用并激活精准降落后，将在着陆过程中提供最终定位，而不仅仅是通过GPS数据。如果着陆目标丢失（即没有发生相对于它的位置更新），如果PLND_STRICT设置得当，它将尝试爬升到PLND_RET_BEHAVE确定的位置。然后，它将尝试再次下降，希望看到着陆目标。它将继续执行此操作，直到发生PLND_RET_MAX重试。然后，根据PLND_STRICT，它要么只是着陆，要么会保持在空中盘旋。

除非设置了PLND_OPTIONS位1，否则飞行员在着陆期间手动重新定位将中止着陆。必要时，最终着陆速度可降至LAND_SPEED以下，以确保精确着陆。通过设置PLND_OPTIONS位2，可以禁用此功能以获得更快的最终着陆速度

1. 四旋翼无人机利用IR-LOCK进行精准降落与精准悬停

1.1 IR-LOCK工作原理

IR-LOCK精准降落系统的工作原理可以概括为以下几个步骤：

  • 信号识别： 当无人机进入着陆区域后，其底部的IR-LOCK传感器开始扫描地面上的红外信标的信号。一旦识别到预设的红外信标信号，传感器会立即锁定该信号。

  • 位置计算： 通过测量红外信标在传感器视野中的位置和相对速度，并结合无人机的当前飞行姿态和高度信息，飞控系统可以计算出无人机与红外信标之间的相对位置关系。

  • 飞行控制： 根据计算出的相对位置关系，飞控系统会对无人机的飞行姿态和高度进行微调，使无人机逐渐接近红外信标所在的位置。在接近过程中，飞控系统会不断接收IR-LOCK传感器传输的实时信号，并根据信号变化动态调整飞行参数。

  • 精准着陆： 当无人机接近到预定高度时，飞控系统会控制无人机缓慢下降并不断修正以精确对准红外信标所在位置实现精准着陆。

IR-LOCK摄像头向飞控发送的数据如下：

Bytes 16-bit words Description
0，1 0 sync (0xaa55)
2，3 1 checksum (sum of all 16-bit words 2-6)
4，5 2 signature number
6，7 3 x center of object
8，9 4 y center of object
10，11 5 width of object
12，13 6 height of object

官方文档

其主要意思为：IR-LOCK传感器附带的默认镜头是3.6mm镜头。使用此镜头，水平视场约为60度，垂直视场约为35度。

IR-LOCK所使用的代码请见附件“irlock.cpp”。

IR-LOCK有关的主要参数有：

  • Heal： 当值不为1时，意味着飞控与IR-LOCK传感器之间可能存在通信问题。

  • TAcq(Target Acquired)： 当值不为1时，表示传感器未看到目标。

  • pX, pY： 显示了从无人机到目标的水平距离。

  • vX、vY： 显示了红外信标相对于无人机的估计速度。

其中

  • GPS定位系统：为飞行器提供米级的定位数据，用以辅助定点飞行。

  • 测距仪：提供飞行器与地面之间的高度信息，增强垂直着陆的精准度。

注意！在使用IR-LOCK进行精准降落的时候一定要使用一个向下的测距仪（RangeFinder），否则该系统无法正常使用，原因是官方源码中开始使用的前提是必须有一个测距仪才能开始用。

1.3 精准降落参数说明

  • PLND_ENABLED设置为1以启用此功能，在写入参数后需要刷新参数表才能查看以下参数。

  • PLND_TYPE设置着陆位置源、IR-LOCK、MAVLink LaANDING_TARGET消息、SITL或GAZEBO模拟的类型。设置为0以禁用。

  • PLND_XY_DIST_MAX如果检测到着陆目标并且距离超过这么多米，车辆将不会开始下降。将0设置为始终下降。

  • PLND_STRICT如果目标丢失，车辆应如何严格降落在目标上

  • PLND_TIMEOUT即使目标丢失，车辆继续下降的时间。在此时间段之后，车辆将根据PLND_STRICT参数所选择的方案尝试着陆重试。

  • PLND_RET_BEHAVE重试时如何移动到最后一个有效着陆目标

    • 值0：前往最后一个检测到着陆目标的位置
    • 值1：朝向探测到的着陆目标的大致位置前进

  • PLND_ALT_MIN即使目标在此高度以下丢失，飞行器也将继续垂直着陆。这需要测距仪才能工作。设置为零可禁用此功能。

  • PLND_ALT_MAX如果找不到目标，飞行器将继续垂直着陆，直到达到这个高度。在此高度以下，如果找不到着陆目标，可能会尝试着陆重试/故障保护。这需要测距仪才能工作。设置为零可禁用此功能。

  • PLND_OPTIONS如果着陆目标正在移动，则设置位0，设置位1以允许在手动重新定位后继续精确着陆，设置位2在最后下降时保持高速。

    • 值0：着陆目标可移动
    • 值1：允许手动重新定位后精确降落
    • 值2：在最后的降落中保持相对高的速度

1.4 技术缺陷

1. 依赖红外信标（需主动部署）

  • 缺点：必须提前部署地面红外信标，且信标需持续供电（如电池或外接电源）。
  • 影响：
    • 不适合突发任务或野外无准备环境。
    • 信标故障（如电量耗尽、遮挡）会导致系统失效。

2. 环境光干扰（如阳光、热源）

  • 缺点：红外传感器易受强红外辐射干扰（如阳光直射、高温地面、其他红外光源）。
  • 影响：
    • 白天户外使用时，信标可能被环境噪声淹没，导致识别距离缩短或失效。
    • 需使用 调制红外信号（如 PWM 编码）来抗干扰，但增加硬件复杂度。

3. 作用距离有限

  • 缺点：红外信标的有效探测距离通常较短（一般 10~30 米，依赖信标功率和传感器灵敏度）。
  • 影响：
    • 无人机需先通过其他方式（如 GPS、视觉）粗对准信标区域，才能进入 IR-LOCK 工作范围。
    • 高空或高速降落场景中可能来不及修正轨迹。

4. 无法提供方位信息（偏航角控制）

  • 缺点：IR-LOCK 仅提供 水平位置偏移量（X/Y 坐标），无法识别信标方向或无人机的机头朝向。
  • 影响：
    • 无人机可能以任意偏航角降落（如机头方向随机），不适合需固定朝向的任务（如舰载无人机需对齐甲板）。
    • 需额外依赖磁力计或视觉系统辅助定向。

5. 动态稳定性不足

  • 缺点：IR-LOCK 在无人机剧烈晃动或高速移动时，可能因图像模糊或信标短暂丢失导致控制不稳定。
  • 影响：
    • 需配合飞控的 动态滤波算法（如卡尔曼滤波）平滑数据，否则易引发震荡。
    • 不适用于高速无人机或强风环境下的精准降落。

6. 多机协同限制

  • 缺点：同一区域内多个无人机共用相同信标时，可能因信号冲突导致混淆。
  • 影响：
    • 需为每台无人机分配独立编码的信标，增加系统复杂度。
    • 多机编队降落时需严格规划信标布局。

7. 无地形感知能力

  • 缺点：IR-LOCK 仅关注信标位置，无法检测降落区域的 地形安全性（如凹凸、障碍物）。
  • 影响：
    • 若信标放置在不平整或危险区域（如斜坡、水面），无人机可能强行降落导致事故。
    • 需结合 超声波/激光雷达 或视觉避障系统补充检测。

8. 硬件安装限制

  • 缺点：红外传感器需安装在无人机底部且无遮挡，对机体结构有要求。
  • 影响：
    • 某些机型（如垂直起降固定翼）可能难以兼容。
    • 传感器镜头污染（如灰尘、水雾）会降低性能。

2. 四旋翼无人机利用OpenMV进行精准降落与精准跟随

2.1 工作原理

openmv是一款低成本的图像处理模块，可以轻松实现常用的图像处理，串口通讯功能，刚接触openmv时我们可以只关心函数的作用而不去了解底层算法实现，简化我们的步骤。openmv可以识别apriltag标签，并测算其位于镜头中的位置。openmv在本项目中，仅充当与降落位置有关信息的传输，也就是通过串口通信将数据传回飞控，而具体引导降落的逻辑则是飞控去完成。openmv充当传感器的作用，所以是可以替代的，比如官方的IRLOCK模块等。

与二维码相比，AprilTags可在更远距离、较差光线和更扭曲的图像环境下被检测到。AprilTags可应对所有种类的图像失真问题，而二维码并不能。也就是说，AprilTags只能将数字ID编码作为其有效载荷。

在本系统中代码（完整代码见下文）的核心逻辑如下：

  • 识别Apriltag： 使用OpenMV Cam的图像处理功能来识别和定位AprilTag标签。当检测到有效标签时，会获取标签的中心点（ cx(), cy() ）、宽度（w）和高度（h）。

  • 计算标签位置： 将标签的中心点坐标从像素单位转换为视场角（FOV）单位，这通过除以图像宽度和高度并乘以相应的视野角来实现。使用标签的黑色边框尺寸和检测到的像素大小，计算标签与摄像头之间的实际距离（毫米）。

  • 发送MAVLink消息： 使用send_landing_target_packet函数将标签的位置和距离信息封装成MAVLink消息。这个函数首先计算一些必要的参数，如目标点的水平和垂直视场角偏差、目标距离（限制在最小和最大距离之间），然后将这些信息以及包序列号和其他元数据打包成MAVLink消息格式。消息中包含的信息包括：时间戳（未使用，设为0）、目标点的X和Y视场角偏差、目标距离、速度（未使用，设为0）、角度（未使用，设为0）、目标点框架（MAV_FRAME_BODY_NED）以及额外的CRC校验值。

  • MAVLink消息校验： 使用checksum函数对打包后的MAVLink消息进行CRC校验，以确保数据在传输过程中的完整性和正确性。

  • 通过UART发送数据： 将包含AprilTag位置和距离信息的MAVLink消息通过UART接口发送给飞控。控制器根据这些信息计算飞行器的控制指令，以实现向目标标签的精准降落。

  • 控制器执行降落： 飞控接收到MAVLink消息后，解析出目标点的位置和距离信息，然后根据这些信息计算飞行器的姿态和速度调整量，以控制飞行器飞向并降落在指定的AprilTag标签上。

2.4 精准降落参数说明

  1. PLND_YAW_ALIGN
  • 说明：Yaw angle from body x-axis to sensor x-axis。
  • 作用：控制无人机的航向角，确保飞行器的机头朝向二维码坐标系的正方向。
  1. PLND_ACC_P_NSE
  • 说明：Kalman Filter Accelerometer Noise, higher values weight the input from the camera more, accels less。
  • 作用：控制加速度计噪声的影响，较高的值意味着飞控系统更多依赖相机数据而非加速度计数据。
  1. PLND_ALT_MAX
  • 说明：Vehicle will continue landing vertically until this height if target is not found. Below this height if landing target is not found, landing retry/failsafe might be attempted. This needs a rangefinder to work. Set to zero to disable this.
  • 作用：当目标丢失时，无人机会继续垂直降落，直到设定的最大高度。如果低于此高度且目标丢失，可能会触发着陆重试或故障保护。此功能需要一个测距传感器。
  1. PLND_ALT_MIN
  • 说明：Vehicle will continue landing vertically even if target is lost below this height. This needs a rangefinder to work. Set to zero to disable this.
  • 作用：在目标丢失后，无人机会继续垂直降落，直到达到设定的最小高度。此功能同样需要一个测距传感器。
  1. PLND_BUS
  • 说明：Precland sensor bus for I2C sensors.
  • 作用：指定精确着陆使用的I2C传感器总线。
  1. PLND_CAM_POS_X
  • 说明：X position of the camera in body frame. Positive X is forward of the origin.
  • 作用：摄像头在机体坐标系中的X位置。正X方向是机体原点前方。
  1. PLND_CAM_POS_Y
  • 说明：Y position of the camera in body frame. Positive Y is to the right of the origin.
  • 作用：摄像头在机体坐标系中的Y位置。正Y方向是机体原点右侧。
  1. PLND_CAM_POS_Z
  • 说明：Z position of the camera in body frame. Positive Z is down from the origin.
  • 作用：摄像头在机体坐标系中的Z位置。正Z方向是机体原点下方。
  1. PLND_EST_TYPE
  • 说明：Specifies the estimation method to be used.
  • 作用：指定用于估算着陆目标的位置的算法或方法。
  1. PLND_LAG
  • 说明：Precision Landing sensor lag, to cope with variable landing_target latency.
  • 作用：精确着陆传感器延迟，用于处理目标检测延迟的变化。
  1. PLND_LAND_OFS_X
  • 说明：Desired landing position of the camera forward of the target in vehicle body frame.
  • 作用：在机体坐标系中，摄像头期望着陆位置相对于目标的X轴偏移。
  1. PLND_LAND_OFS_Y
  • 说明：Desired landing position of the camera right of the target in vehicle body frame.
  • 作用：在机体坐标系中，摄像头期望着陆位置相对于目标的Y轴偏移。
  1. PLND_LAND_OFS_Z
  • 说明：Desired landing position of the camera downward of the target in vehicle body frame.
  • 作用：在机体坐标系中，摄像头期望着陆位置相对于目标的Z轴偏移。
  1. PLND_OPTIONS
  • 说明：Precision Landing Extra Options.
  • 作用：精确着陆的附加选项，用于进一步控制着陆行为。
  1. PLND_RET_BEHAVE
  • 说明：Precision Land will do the action selected by this parameter if a retry to a landing is needed.
  • 作用：如果需要重试着陆，选择在重试时要执行的操作。
  1. PLND_RET_MAX
  • 说明：Precision Landing Maximum number of retries for a failed landing. Set to zero to disable landing retry.
  • 作用：设置着陆失败后最大重试次数，设置为0表示禁用重试。
  1. PLND_STRICT
  • 说明：How strictly should the vehicle land on the target if target is lost.
  • 作用：如果目标丢失，飞控系统在着陆时的严格程度。设置为1表示严格执行精准着陆，设置为0则较宽松。
  1. PLND_TIMEOUT
  • 说明：Time for which vehicle continues descend even if target is lost. After this time period, vehicle will attempt a landing retry depending on PLND_STRICT parameter.
  • 作用：当目标丢失时，无人机会继续降落的最大时间。超时后，飞控根据 PLND_STRICT 参数决定是否进行重试。
  1. PLND_TYPE
  • 说明：Precision Land Type.
  • 作用：指定精准着陆的类型，决定飞控使用何种方法进行降落，如基于视觉、雷达或结合多种传感器。
  1. PLND_XY_DIST_MAX
  • 说明：The vehicle will not start descending if the landing target is detected and it is further than this many meters away. Set 0 to always descend.
  • 作用：设置目标距离无人机的最大水平距离。若目标超过此距离，无人机不会开始降落。设置为0表示无论距离多远都可以开始降落。

2.5 检查是否正常工作

在将所有参数写入飞控后，正确连接飞控与open MV，将飞控接电，应该看到open MV摄像头红灯亮起并闪烁，这意味着它并没有看到任何Apriltag，当我们把生成的Apriltag放到摄像头前合适位置时，会发现open MV呈绿灯闪烁。此时去地面站查看，在“飞行数据”界面，按下Ctrl+F，选择“MAVLink Inspector”，若正确识别并向飞控发送数据，会在此窗口生成两个84Hz的消息，LANDING_TARGET和DISTANCE_SENSOR。如果可以收到这两个消息，那证明和飞控连接没有问题。

注：无论是OpenMV还是IR-LOCK，都需要将本系统的摄像头焦距调整好，要保证其视野清晰，可以先手拿飞行器进行测试，看看最远检测到的距离是多少，再进行微调
